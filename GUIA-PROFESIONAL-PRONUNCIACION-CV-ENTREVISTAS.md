## 🎤 GUÍA DE PRONUNCIACIÓN (AMERICAN ENGLISH)

### DIFERENCIAS CLAVE: BRITISH vs AMERICAN

```
BRITISH vs AMERICAN:

PRONUNCIACIÓN DE -R:
British: "car" = CAH (no pronuncias la r)
American: "car" = CAR (sí pronuncias la r)
→ En tu estudio: PRONUNCIA LA R (American English)

VOCALES:
British: "dance" = DAHNS
American: "dance" = DANS
→ En tu estudio: PRONUNCIA LA A como en "cat"

ESTRÉS EN PALABRAS:
British: "schedule" = SHED-yool
American: "schedule" = SKED-jool
→ En tu estudio: SKED-jool

CONSONANTES:
T entre vocales:
British: "butter" = BUTTA (suave)
American: "butter" = BUDDER (casi como d)
→ En tu estudio: Pronuncia la T claramente
```

## ✉️ EMAIL TEMPLATES PROFESIONALES

### TEMPLATE 1: Presenting your work in a meeting

```
SITUACIÓN: Presentas tu proyecto en una reunión

"Good morning everyone. Today I want to present the results of our 
X-week project on [tema]. 

We had three main objectives:
1. [Objetivo 1]
2. [Objetivo 2]
3. [Objetivo 3]

I'm pleased to report that we achieved [resultado]. The key metrics show 
a [número]% improvement in [métrica]. 

However, we did encounter some challenges with [desafío]. To address this, 
we implemented [solución], which resulted in [resultado].

Moving forward, our next steps are:
- [Acción 1]
- [Acción 2]
- [Acción 3]

I'd be happy to discuss any questions you might have."

PALABRAS CLAVE:
- I'm pleased to report = Estoy complacido de reportar
- We achieved = Logramos
- However = Sin embargo
- Moving forward = Adelante
- I'd be happy to = Estaría feliz de
```

### TEMPLATE 2: Asking for help/feedback

```
SITUACIÓN: Solicitas ayuda o feedback

"Hi [Name],

I wanted to reach out because I'm working on [proyecto] and could use 
your expertise in [área]. 

Specifically, I'm trying to [objetivo], but I've run into an issue with 
[problema]. 

Would you have time this week to review [archivo/idea]? I'm aiming to 
[meta] by [fecha], so any feedback would be greatly appreciated.

Thanks for your time!

[Tu nombre]"

PALABRAS CLAVE:
- reach out = comunicarme
- could use your expertise = podría usar tu experiencia
- run into = tropezarme con
- Would you have time = ¿Tendrías tiempo?
- greatly appreciated = muy apreciado
```

### TEMPLATE 3: Proposing a technical solution

```
SITUACIÓN: Propones una solución técnica

"After analyzing the current system, I recommend implementing [solución] 
for the following reasons:

1. Performance: [Explicar mejora]
2. Maintainability: [Explicar mantenimiento]
3. Scalability: [Explicar escalabilidad]

The implementation would involve:
- [Paso 1]
- [Paso 2]
- [Paso 3]

Timeline: [Duración estimada]
Resources needed: [Qué recursos]

I estimate this would reduce [problema] by approximately [número]%, 
resulting in [beneficio].

Please let me know if you'd like me to elaborate on any aspect."

PALABRAS CLAVE:
- After analyzing = Después de analizar
- I recommend = Recomiendo
- implement = implementar
- involve = implicar
- reduce = reducir
- elaborate = elaborar
```

### TEMPLATE 5: Negotiating a freelance rate

```
SITUACIÓN: Negocias una tarifa de freelance

"Hi [Cliente],

Thank you for reaching out about the [proyecto] opportunity. I'm very 
interested in working with you.

Based on my experience with [tipo de proyectos similares] and the scope 
of work you described, my rate is [USD/hora] for [tipo de trabajo].

This includes:
- [Deliverable 1]
- [Deliverable 2]
- [Número] hours of revisions

For a project of this scope, I estimate a timeline of [X] weeks.

I'm flexible on rate depending on:
- Long-term engagement (recurring work)
- Project complexity
- Timeline urgency

Would this work for you? I'm happy to discuss any questions."

PALABRAS CLAVE:
- reaching out = comunicarse
- based on my experience = basado en mi experiencia
- scope of work = alcance del trabajo
- deliverable = entregable
- flexible = flexible
- long-term engagement = compromiso a largo plazo
```

## 🎓 EXPRESIONES TÉCNICAS COMUNES

### En REUNIONES:

```
"Let me circle back on that" 
= Déjame volver a eso
Uso: Cuando quieres continuar una discusión después

"Can we take this offline?"
= ¿Podemos hablar de esto fuera de esta reunión?
Uso: Para discusiones complejas que toman mucho tiempo

"I'd like to dive deeper into that"
= Me gustaría profundizar en eso
Uso: Para entrar en más detalles

"Let's table this for now"
= Aparquemos esto por ahora
Uso: Para posponer un tema

"What's the ask here?"
= ¿Cuál es la solicitud aquí?
Uso: Para clarificar qué se espera de ti

"I'll take an action on that"
= Yo me haré cargo de eso
Uso: Para asumir una tarea

"Let me action that"
= Déjame ejecutar eso
Uso: Para confirmar que harás algo
```

### En CODE REVIEWS:

```
"Can you refactor this?"
= ¿Puedes refactorizar esto?
Uso: Pedir que reorganices código

"This violates [principio]"
= Esto viola [principio]
Uso: Cuando el código no sigue buenas prácticas

"Consider using [patrón] instead"
= Considera usar [patrón] en su lugar
Uso: Sugerencia de mejora

"This could introduce a race condition"
= Esto podría introducir una condición de carrera
Uso: Problema de concurrencia

"LGTM: Looks Good To Me"
= Se ve bien
Uso: Aprobación de código

"Let's add test coverage for this"
= Agreguemos pruebas para esto
Uso: Pedir tests adicionales

"This is coupled too tightly"
= Esto está demasiado acoplado
Uso: Cuando hay dependencias fuertes
```

### En DOCUMENTACIÓN:

```
"Describe the rationale for this approach"
= Describe la razón de este enfoque
Uso: Explicar por qué elegiste algo

"This needs clarification"
= Esto necesita aclaración
Uso: Cuando algo no está claro

"Can you add an example?"
= ¿Puedes agregar un ejemplo?
Uso: Para que sea más comprensible

"See [sección] for more details"
= Ver [sección] para más detalles
Uso: Referencia cruzada

"As mentioned in [documento]"
= Como se menciona en [documento]
Uso: Referencia a documentación anterior
```

## 🎤 ENTREVISTA TÉCNICA EN INGLÉS

### PREGUNTAS TÍPICAS Y RESPUESTAS

#### Q1: "Tell me about yourself"

```
ESTRUCTURA:
"I'm a [rol] with [X] years of experience in [especialidad]. 
Most recently, I worked at [empresa] where I [logro principal]. 

My expertise is in [tech stack], and I'm passionate about [tema]. 
I'm interested in this role because [razón específica para esta empresa]."

EJEMPLO REAL:
"I'm a DevOps Engineer with 4 years of experience in cloud infrastructure 
and automation. Most recently, I worked at [Empresa] where I architected 
and deployed a Kubernetes-based microservices platform that reduced 
deployment time by 60%.

My expertise is in containerization, CI/CD pipelines, and infrastructure 
as code. I'm passionate about building reliable, scalable systems.

I'm interested in this role at [Empresa] because I've followed your work 
in [tema] and I'm impressed by your approach to [aspecto específico]."

TIEMPO: 60-90 segundos
```

#### Q2: "Why are you interested in this role?"

```
DO:
✓ Menciona algo específico de la empresa
✓ Conecta tus skills con sus necesidades
✓ Muestra entusiasmo genuino

DON'T:
✗ "I need a job" (muy obvio)
✗ "The salary is good" (evita dinero)
✗ Información incorrecta sobre la empresa

EJEMPLO:
"I'm excited about this opportunity because [Empresa] is building 
state-of-the-art solutions in [área]. Specifically, I'm interested in 
how you're using [tecnología] to solve [problema].

Given my background in [especialidad], I believe I can contribute by 
[cómo contribuirías]. Plus, I'm eager to work with [equipo/expertos] 
to learn and grow."
```

#### Q3: "Describe a technical challenge you faced"

```
ESTRUCTURA (STAR method):
1. SITUATION: "At [empresa], we were..."
2. TASK: "The challenge was..."
3. ACTION: "I took the following steps: [paso1], [paso2], [paso3]"
4. RESULT: "As a result, we achieved [resultado cuantificable]"

EJEMPLO:
"At [Empresa], we had a microservices system that was experiencing 
cascading failures. When one service went down, it would bring down 
dependent services.

I architected and implemented a circuit breaker pattern and added 
comprehensive monitoring. I also set up automated health checks and 
graceful degradation.

As a result, system availability improved from 98% to 99.9%, and MTTR 
(mean time to recovery) dropped from 45 minutes to 5 minutes."

TIEMPO: 2-3 minutos
```

#### Q4: "What's your biggest weakness?"

```
ESTRUCTURA:
"My initial [weakness] was [specific ejemplo]. 
However, I recognized this and [acción que tomaste]. 
Now [progreso medible]."

EJEMPLO:
"Early in my career, I struggled with delegation because I wanted to 
ensure everything was perfect. However, I realized this limited the 
team's growth.

I started mentoring junior engineers, which initially required me to 
loosen my standards slightly. But I found this actually improved quality 
because the team took ownership. Now I'm comfortable delegating complex 
tasks and have mentored 3 junior developers into mid-level positions."

NOTA: Elige una "weakness" que sea realista pero que hayas mejorado
```

#### Q5: "Do you have any questions for us?"

```
PREGUNTAS INTELIGENTES:

✓ "What does success look like in this role in the first 90 days?"
✓ "How do you measure engineering productivity on this team?"
✓ "What's the biggest technical debt or challenge the team is facing?"
✓ "Can you tell me about the tech stack and architecture decisions?"
✓ "How does the team handle on-call and incident response?"
✓ "What's the culture around code reviews and technical feedback?"
✓ "How are decisions made about new technology adoption?"
✓ "What's the team structure and how many engineers report to this role?"

PREGUNTAS EVITAR:
✗ "How much vacation do I get?" (pregunta al final)
✗ "When can I work from home?" (demasiado pronto)
✗ "How soon can I be promoted?" (pareces ambicioso pero inseguro)
```

## 🌍 FREELANCE Y TRABAJO REMOTO EN INGLÉS

### PLATAFORMAS PARA ENCONTRAR TRABAJO:

```
1. UPWORK (upwork.com)
   - Freelancers compiten por proyectos
   - Comisión: 5-20% (depende de relación con cliente)
   - Proyectos: $500-5,000+ por proyecto
   - Tiempo: Corto plazo típicamente

2. TOPTAL (toptal.com)
   - Vetted developers (debes pasar entrevista técnica)
   - Mejor: Clientes de alta calidad, salarios $60-150+/hora
   - Pros: Solo clientes serios, sin subastas de precios
   - Requisito: English B1+ (verificado)

3. HIRED (hired.com)
   - Companies buscan a ti (no compites)
   - Rango: $60-250+/hora
   - Mejor: Menos trabajo de marketing

4. STACKOVERFLOW JOBS (stackoverflow.com/jobs)
   - Más empresa que freelance
   - Rango: $50-150+/hora

5. GURU (guru.com)
   - Similar a Upwork
   - Comisión: 8.5-12.5%
   - Rango: Variable

6. FREELANCER (freelancer.com)
   - Similar a Upwork
   - Comisión: 10-30%
```

### ESCRIBIR TU PERFIL EN UPWORK/TOPTAL:

```
TÍTULO (5-7 palabras):
"Senior DevOps Engineer | Kubernetes | CI/CD | AWS"
(NO: "Engineer seeking remote work")

DESCRIPCIÓN (150-200 palabras):
"I'm a DevOps engineer with 4+ years of experience building, deploying, 
and maintaining cloud infrastructure at scale.

EXPERTISE:
• Kubernetes orchestration and management
• CI/CD pipeline design and implementation (Jenkins, GitLab CI)
• Infrastructure as Code (Terraform, Ansible)
• AWS services (EC2, RDS, S3, Lambda, CloudFormation)

• Monitoring and logging (Prometheus, ELK Stack)

EXPERIENCE:
I've worked with startups and mid-size companies, helping them scale

PREVIOUS PROJECTS:
• Migrated 50+ microservices to Kubernetes
• Reduced deployment time from 2 hours to 5 minutes
• Saved $20k/month through cloud cost optimization

I'm committed to delivering reliable infrastructure and excellent

PALABRAS CLAVE:
✓ Cuantifica logros ($, tiempo, %)
```
